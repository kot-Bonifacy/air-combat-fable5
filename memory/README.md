# memory/ — konwencja zapisu

Pamięć projektu między sesjami vibecodingu. Zasady:

- `MEMORY.md` — indeks: jedna linia na wpis, format `- [tytuł](plik.md) — jednozdaniowy hook`.
- `project_phaseN_decisions.md` — po **zakończeniu** fazy N: decyzje nieoczywiste z kodu,
  pułapki, na które się natknęliśmy, i rzeczy celowo odłożone. Nie powtarzamy tego,
  co widać w kodzie, CLAUDE.md ani w `docs/phases/faza-NN.md`.
- Daty zawsze absolutne (np. 2026-06-10), nigdy „wczoraj/ostatnio".
- Wpis nieaktualny → poprawiamy lub usuwamy, nie dopisujemy sprostowań na końcu.
